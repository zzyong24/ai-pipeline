# Prompt 模板：重构

## 适用场景

需要重构现有代码（抽取函数、合并重复、改变架构等）时使用。

---

## 模板

```
需要重构 {{PROJECT_NAME}} 项目中的某段代码。

## 重构目标
{清晰描述要达到的效果}
示例：
- 拆分过长的 XXX 函数
- 消除 A、B、C 三处重复的业务逻辑
- 把 XXX 功能从 Service 层移到专门的 Helper

## 约束
- [ ] 不改变外部行为（签名/返回值不变）
- [ ] 不破坏已有测试
- [ ] 保持与现有代码风格一致

## 参考文件
- `.aies/spec/quality-gates.md` → 避免引入反模式
- `.ai/index.md` → 定位所有调用方

## 执行流程
1. Phase 1：输出重构计划（before/after 对比 + 影响范围）
2. 等用户确认后再执行修改
3. 修改后列出所有被影响的文件
4. Phase 3：过质量自检清单 + 建议跑测试
```

---

## 重构前 AI 必须输出

```markdown
## 重构计划

### Before
- 当前代码结构：...
- 问题：...

### After
- 目标结构：...
- 改进：...

### 影响范围
- 直接修改：{文件列表}
- 调用方影响：{调用方列表 + 是否需要同步修改}
- 测试影响：{相关测试文件}

### 风险评估
- 🔴 高风险点：...
- 🟠 中风险点：...
- 🟡 低风险点：...

### 是否可以开始？（等用户确认）
```

---

## 重构红线

- ❌ 不要大规模重构用户未明确要求的代码
- ❌ 不要在一次重构中同时改变结构和行为
- ❌ 不要删除"看起来没用"的代码而不验证
- ✅ 每次只做一件事（单一职责）
- ✅ 保留 git 历史可追溯（小步提交）
